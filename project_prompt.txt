- Maintains persistent identity using vector embeddings
- Analyzes behavior, pose, and threats (weapons, falls)
- Provides context-aware alerts (not raw detections)
- Exposes a clean REST + WebSocket API for a frontend dashboard

## 🧩 MODULES TO INTEGRATE

Integrate each module as a **separate Python file** inside a `modules/` directory. Each module must expose a clean function interface that the main pipeline can call.

### 1. 🔑 Embedding & Re-ID Engine (`modules/embedding_engine.py`)

**Purpose:** Convert every detected person/object crop into a 512-dim vector embedding. Use this for persistent identity across frames and cameras.

**Requirements:**
- Use a pretrained **MobileNetV2** (via PyTorch/torchvision) as the backbone, remove the classification head, output a 512-dim L2-normalized embedding.
- Maintain an in-memory **identity registry** (dict): maps `global_id → {embedding, metadata, history}`.
- On each new detection crop:
  1. Generate embedding.
  2. Compare via **cosine similarity** against all known embeddings.
  3. If similarity > 0.82 → assign existing `global_id`.
  4. If no match → create new `global_id` (e.g., `Person_001`).
- Each identity entry stores:
```json
{
  "global_id": "Person_001",
  "embedding": [...],
  "metadata": {
    "face_name": null,
    "activity": "walking",
    "risk_level": "low",
    "clothing_color": "blue",
    "last_seen_camera": "CAM_01",
    "last_seen_time": "2024-01-15T10:34:00Z"
  },
  "history": [
    {"time": "...", "camera": "...", "activity": "...", "location": "..."}
  ]
}
```
- Expose functions: `get_or_create_identity(crop_img) → global_id`, `update_identity_metadata(global_id, updates)`, `search_similar(query_embedding, top_k=5) → [global_id, ...]`, `get_all_identities() → list`, `get_identity(global_id) → dict`.

**Optional:** Persist registry to SQLite using `modules/database.py` (see below).

### 2. 🙂 Face Recognition Module (`modules/face_recognition_module.py`)

**Purpose:** Link a tracked person's `global_id` to a real name if their face matches a known database.

**Requirements:**
- Use the **`face_recognition`** library (or **DeepFace** as fallback).
- Maintain a local face database: `data/known_faces/` folder where each subfolder is named after the person and contains their face images.
- On detection:
  1. Crop face region from bounding box.
  2. Run face encoding.
  3. Compare against known encodings.
  4. If match found (tolerance ≤ 0.5) → set `metadata.face_name` on the identity.
- Expose: `load_known_faces(faces_dir) → encodings_dict`, `recognize_face(face_crop) → name | None`.
- This module is **optional per detection** — only run if a face is clearly visible (bounding box height > 80px).

---

### 3. 🦴 Pose & Activity Analysis (`modules/pose_analyzer.py`)

**Purpose:** Detect human skeleton, classify activity, detect falls.

**Requirements:**
- Use **YOLOv8-Pose** (`yolov8n-pose.pt`) — this integrates naturally with your existing YOLOv8 setup.
- From keypoints, classify activity into: `walking`, `running`, `standing`, `sitting`, `falling`, `fighting`, `crouching`.
- **Fall Detection Logic:**
  - If head keypoint Y-position is below hip keypoint Y-position (inverted) AND confidence > 0.6 → flag as `falling`.
  - Trigger an alert immediately.
- **Activity Classification Rules (heuristic-based):**
  - Compare keypoint velocities across 3 frames (use a rolling buffer per `global_id`).
  - High velocity + upright → `running`. Low velocity + upright → `walking`. Keypoints compressed vertically → `sitting`.
- Expose: `analyze_pose(frame, bbox) → {activity, keypoints, fall_detected, confidence}`.
- Update `metadata.activity` on the identity registry.

---

### 4. 🔪 Weapon Detection Module (`modules/weapon_detector.py`)

**Purpose:** Detect knives and firearms in the scene and link them to the nearest tracked person.

**Requirements:**
- Use a **fine-tuned YOLOv8** model for weapon detection. If no custom model is available, use the standard COCO-trained YOLOv8 which detects class `"knife"` — note this has limited recall.
- Recommended: Download a public weapon-detection YOLOv8 weights file (e.g., from Roboflow Universe — `weapon-detection` dataset).
- After detection:
  1. Get weapon bounding box center.
  2. Find the nearest tracked person bounding box center (Euclidean distance).
  3. Assign weapon to that person's `global_id`.
  4. Set `metadata.risk_level = "critical"` and trigger alert.
- Expose: `detect_weapons(frame) → [{weapon_type, bbox, confidence, nearest_person_id}]`.

---

### 5. 🏃 Motion & Anomaly Detection (`modules/motion_detector.py`)

**Purpose:** Detect motion in otherwise empty zones, and flag loitering or unusual stillness.

**Requirements:**
- Use **OpenCV Background Subtraction** (`cv2.createBackgroundSubtractorMOG2`).
- Define **zones** (configurable rectangles per camera) — e.g., restricted areas.
- If motion detected inside a restricted zone → trigger zone-breach alert.
- **Loitering detection:** If same `global_id` stays in the same bounding box region for > N seconds (configurable, default 30s) → flag as loitering.
- Expose: `detect_motion(frame, zones) → {motion_detected, active_zones, motion_mask}`, `check_loitering(global_id, bbox, timestamp) → bool`.

---

### 6. 🚨 Alert Engine (`modules/alert_engine.py`)

**Purpose:** Generate context-aware, actionable alerts — not raw detections.

**Requirements:**
- Alerts must be **structured and human-readable**, not just "weapon detected".
- Alert schema:
```json
{
  "alert_id": "uuid",
  "severity": "critical | high | medium | low",
  "type": "weapon | fall | loitering | zone_breach | unknown_person",
  "message": "John Doe (Person_001) is holding a knife at Camera 2, Zone A",
  "global_id": "Person_001",
  "camera_id": "CAM_02",
  "location": "Zone A",
  "timestamp": "2024-01-15T10:34:00Z",
  "thumbnail_path": "data/thumbnails/alert_uuid.jpg",
  "acknowledged": false
}
```
- Store alerts in SQLite via `modules/database.py`.
- Push alerts in real-time over the **WebSocket** endpoint.
- Expose: `create_alert(type, global_id, camera_id, details) → alert`, `acknowledge_alert(alert_id)`, `get_recent_alerts(limit=50) → list`.
- **Deduplication:** Don't re-fire the same alert for the same `global_id` + `type` within 60 seconds.

### 7. 🗄️ Database Layer (`modules/database.py`)

**Purpose:** Persist identities, alerts, and events using SQLite (lightweight, no external DB needed).

**Requirements:**
- Use Python's built-in `sqlite3` or **SQLAlchemy Core** (no ORM needed).
- Tables:

**`identities`**
```sql

  alert_id TEXT PRIMARY KEY,

  acknowledged BOOLEAN DEFAULT 0

```

- Expose: `save_identity(identity_dict)`, `save_event(event_dict)`, `save_alert(alert_dict)`, `get_identity_history(global_id) → list`, `get_alerts(acknowledged=False) → list`.

## 🔄 MAIN PROCESSING PIPELINE (`core/pipeline.py`)

**Purpose:** Orchestrate all modules into a single per-frame processing function.

**The pipeline for each video frame:**

```python
def process_frame(frame, camera_id: str) -> PipelineResult:
    # 1. YOLOv8 detection (existing)
    detections = yolo_model.detect(frame)
    
    # 2. For each detected person bounding box:
    for det in detections.persons:
        crop = frame[det.y1:det.y2, det.x1:det.x2]
        
        # 3. Embedding → Persistent Identity
        global_id = embedding_engine.get_or_create_identity(crop)
        
        # 4. Face Recognition (if face visible)
        if det.has_face():
            name = face_module.recognize_face(crop)
            if name:
                embedding_engine.update_identity_metadata(global_id, {"face_name": name})
        
        # 5. Pose & Activity
        pose_result = pose_analyzer.analyze_pose(frame, det.bbox)
        embedding_engine.update_identity_metadata(global_id, {"activity": pose_result.activity})
        if pose_result.fall_detected:
            alert_engine.create_alert("fall", global_id, camera_id, pose_result)
        
        # 6. Loitering check
        if motion_detector.check_loitering(global_id, det.bbox, current_time):
            alert_engine.create_alert("loitering", global_id, camera_id, {})
    
    # 7. Weapon Detection (on full frame)
    weapons = weapon_detector.detect_weapons(frame)
    for w in weapons:
        alert_engine.create_alert("weapon", w.nearest_person_id, camera_id, w)
    
    # 8. Motion / Zone breach
    motion = motion_detector.detect_motion(frame, zones=camera_zones[camera_id])
    
    # 9. Save events to DB
    database.save_events(...)
    
    # 10. Return annotated frame + metadata
    return PipelineResult(annotated_frame, identities, alerts)
```

## 🚀 INTEGRATION INSTRUCTIONS (STEP BY STEP)

Follow this order to integrate without breaking the existing backend:

**Step 1 — Setup structure**
Create the `modules/`, `routers/`, `models/`, `data/`, `weights/` directories.

**Step 2 — Database first**
Implement `modules/database.py` and run `database.init_db()` on app startup. This is independent of everything else.

**Step 3 — Embedding engine**
Implement `modules/embedding_engine.py`. Test it standalone with a sample image crop before connecting to the pipeline.

**Step 4 — Wrap existing detection**
Move your current YOLOv8 detection logic into `core/pipeline.py`. Make it call `embedding_engine.get_or_create_identity()` for each person detected.

**Step 5 — Add modules one at a time**
Integrate modules in this order (each is independent): pose_analyzer → motion_detector → weapon_detector → face_recognition_module → alert_engine.

**Step 6 — Add API routes**
Add new routers to your FastAPI app using `app.include_router()`. Don't modify existing routes.

**Step 7 — WebSocket streaming**
Add `/ws/stream/{camera_id}` — encode the annotated frame as JPEG bytes and send via WebSocket on each frame.

**Step 8 — Testing**
Test each module with a sample video file before connecting to live CCTV.
